Git Workflow: O que é na prática?

O termo "Git workflow" refere-se ao conjunto de práticas e processos que uma equipe ou indivíduo segue ao usar o sistema de controle de versão Git para gerenciar e colaborar em projetos de software. O Git é uma ferramenta de controle de versão distribuída amplamente utilizada, e seu workflow proporciona uma estrutura para organizar como as alterações são propostas, revisadas, aceitas e integradas em um projeto.

Clone

Clona um repositório remoto para o seu computador.

git clone [URL]

Branch

Cria e gerencia branches para desenvolver novos recursos.

git branch [nome_da_branch]

Commit

Registra as alterações definitivamente no repositório.

git commit -m "Mensagem do commit"

Merge

Integra as alterações de uma branch ao branch principal.

git merge [nome_do_branch]

Pull Request

Solicita a integração das suas alterações ao branch principal.

git pull-request
